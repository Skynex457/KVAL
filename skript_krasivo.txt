1. НАИМЕНОВАНИЕ И КРАТКОЕ ОПИСАНИЕ
-----------------------------------
Android Studio — официальная интегрированная среда разработки (IDE)
от Google для создания мобильных приложений под операционную систему
Android. Поддерживает языки программирования Kotlin, Java и C++.
Включает в себя редактор кода с подсветкой синтаксиса, эмулятор
устройств Android, отладчик, профилировщик ресурсов и систему сборки
Gradle.

Назначение: разработка, отладка и тестирование Android-приложений
для всех типов устройств (смартфоны, планшеты, умные часы, TV, Auto).

2. ЗАПУСК ПРОГРАММЫ
-------------------
Способы запуска Android Studio:

  2.1. Через графическое меню:
       - Нажмите "Activities" (Супер-клавиша)
       - Введите "Android Studio" в поиске
       - Кликните по иконке приложения

  2.2. Через терминал:
       $ /opt/android-studio/bin/studio.sh
       или
       $ android-studio

  2.3. Через командную строку с указанием проекта:
       $ android-studio /путь/к/проекту

При первом запуске программа предложит импортировать настройки
из предыдущей версии или начать с чистой конфигурации.

3. ЗАПЛАНИРОВАННЫЙ ПРОГРАММОЙ ВЫХОД
-----------------------------------
Для корректного завершения работы используйте один из способов:

  3.1. Стандартный выход:
       Menu → File → Exit (или нажмите Ctrl+Q)

  3.2. Выход с сохранением:
       - Если есть открытые файлы с изменениями, программа предложит
         сохранить их перед выходом
       - Автоматическое сохранение выполняется в папку .idea/saved/

  3.3. Аварийное завершение:
       - Если программа зависла, можно завершить принудительно:
         $ pkill -f studio.sh
       - В этом случае изменения не сохраняются, но есть автосохранение

4. ПЕРЕЧЕНЬ ВЫПОЛНЯЕМЫХ ФУНКЦИЙ
--------------------------------
Android Studio выполняет следующие основные функции:

  4.1. Редактирование кода:
       - Подсветка синтаксиса для Kotlin, Java, XML, C++
       - Автодополнение кода (IntelliSense)
       - Рефакторинг (переименование, извлечение методов)
       - Навигация по коду (Ctrl+Click)
       - Проверка кода в реальном времени

  4.2. Сборка (Build):
       - Компиляция кода в APK (Android Package)
       - Сборка AAB (Android App Bundle)
       - Обработка ресурсов (изображения, макеты)
       - Интеграция с Gradle (система сборки)

  4.3. Эмуляция:
       - Запуск виртуальных Android-устройств
       - Поддержка разных версий Android
       - Эмуляция датчиков (GPS, акселерометр)
       - Эмуляция телефонных звонков и SMS

  4.4. Отладка:
       - Пошаговое выполнение кода
       - Точки останова (Breakpoints)
       - Просмотр значений переменных
       - Отладка реальных устройств через USB

  4.5. Профилирование:
       - Анализ использования CPU
       - Мониторинг памяти (Memory Profiler)
       - Анализ сети (Network Profiler)
       - Мониторинг энергопотребления

  4.6. Управление проектом:
       - Создание новых проектов с шаблонами
       - Импорт существующих проектов
       - Управление версиями (Git, SVN)
       - Управление библиотеками (Maven, Gradle)

5. ОСНОВНЫЕ ПРИЕМЫ РАБОТЫ
--------------------------

  5.1. Создание нового проекта:
       File → New → New Project
       → Выбрать шаблон (например, "Empty Views Activity")
       → Указать имя проекта, пакет, минимальную версию SDK
       → Нажать "Finish"

  5.2. Запуск приложения на эмуляторе:
       Tools → Device Manager → Create device
       → Выбрать модель телефона (Pixel, Nexus и др.)
       → Выбрать образ системы (Android API уровень)
       → Нажать кнопку "▶" (Run)

  5.3. Запуск на реальном устройстве:
       - Включить "Режим разработчика" на телефоне
       - Включить "Отладка по USB"
       - Подключить телефон к ПК через USB-кабель
       - Выбрать устройство в списке при запуске

  5.4. Отладка через точки останова:
       - Кликнуть по номеру строки (появится красный кружок)
       - Нажать кнопку "Debug" (жук) вместо "Run"
       - Использовать клавиши отладки:
         F8 - шаг с обходом (Step Over)
         F7 - шаг с заходом (Step Into)
         F9 - продолжить выполнение (Resume)

  5.5. Работа с Git:
       VCS → Enable Version Control Integration → Выбрать Git
       Для коммита: Ctrl+K
       Для пуша: Ctrl+Shift+K
       Для просмотра истории: VCS → Git → Show History

  5.6. Поиск и замена:
       Поиск по файлу: Ctrl+F
       Поиск по проекту: Ctrl+Shift+F
       Замена: Ctrl+R

  5.7. Горячие клавиши:
       Ctrl+Space    - Автодополнение
       Ctrl+Shift+R  - Запуск приложения
       Alt+Enter     - Быстрые исправления
       Ctrl+Alt+L    - Форматирование кода
       Ctrl+/        - Комментирование строки

6. ТИПОВЫЕ ПРОБЛЕМЫ И ИХ РЕШЕНИЕ
--------------------------------

  6.1. Эмулятор не запускается:
       Причина: отключена виртуализация в BIOS
       Решение: включить VT-x/AMD-V в BIOS
       Проверка: egrep -c '(vmx|svm)' /proc/cpuinfo
       Если 0 - значит виртуализация отключена

  6.2. Gradle sync failed:
       Причина: проблемы с кэшем или библиотеками
       Решение: File → Invalidate Caches / Restart → Invalidate and Restart

  6.3. Не видит реальное устройство:
       Причина: отключена отладка по USB
       Решение: 
       - Проверить настройки телефона
       - Выполнить в терминале: adb kill-server && adb start-server
       - Проверить: adb devices

  6.4. Медленная работа:
       Причина: недостаточно памяти
       Решение: 
       - File → Settings → Appearance & Behavior → System Settings → Memory
       - Увеличить Heap Size до 2048 MB или выше

  6.5. Ошибка при сборке (Build failed):
       Причина: проблемы с зависимостями или синтаксисом
       Решение:
       - Проверить файл build.gradle
       - Выполнить: Build → Clean Project
       - Затем: Build → Rebuild Project
